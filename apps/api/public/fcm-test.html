SoLi FCM Push Test Page

Development tool — authenticates with the backend, generates a browser FCM token, persists it to device_tokens, and tests push delivery.
Open this page from the NestJS server only: http://localhost:3000/fcm-test.html — not Live Server, not file://.

Step 0 — Authenticate not signed in

Sign in to link your FCM token to your account. The token registration endpoint (POST /api/notifications/my/push-tokens) requires a valid session.
Try seeded dev accounts: customer@soli.dev / owner@soli.dev (password set during pnpm seed) or sign up with any email below.

Step 1 — Get FCM Token & Register in DB checking SW…

Gets your browser FCM token from Firebase and registers it in the device_tokens table via POST /api/notifications/my/push-tokens. Requires Step 0. Re-registering the same token refreshes last_seen_at.

Step 2 — Send Test Push via Backend API

Sends a push directly to a raw FCM token via the unauthenticated test endpoint — bypasses device_tokens. Use to verify Firebase delivery works. Token auto-fills from Step 1.

Step 3 — Verify Tokens in device_tokens

Shows all tokens registered for your account (active + inactive). Refreshes automatically after Step 1. Token values are masked to the last 8 characters.
